\subsubsection{Building Block: Authenticated OPRF}
%
\input{authOPRF.tex}

We will use a pseudorandom function (obliviously computed) and we recall the security definition of a PRF in \appref{sec:defn:prf}. We extend the definition to a three party functionality \authOPRFFunc (\figref{fig:authOPRF}), called \emph{authenticated OPRF}, where a group of $\numAuditors$ auditors jointly sign a list of inputs to a pseudorandom function. \authOPRFFunc receives the key for the PRF from the client, who acts as the sender. The receiver of the PRF evaluations is the server, who does not input anything into \authOPRFFunc. 

We realize \authOPRFFunc by leveraging the the 2HashDH PRF construction~\cite{jarecki2014round} in a group where the OM gap-DH problem is hard; $\prf{\prfKeyClient}(\genericFnInput) = \oprfRO[2](\genericFnInput, \oprfRO[1](\genericFnInput)^{\prfKeyClient})$, where $\oprfRO[1], \oprfRO[2]$ are modeled as random oracles. 
In the protocol \protocolAuthOPRF (\figref{fig:authOPRF}), the auditors ``blind'' the blocklist elements $\{ \oprfRO[1](\genericBlocklistElmt[\setIdx])^{\genericRand[\setIdx]'}\}_{\genericBlocklistElmt[\setIdx] \in \blocklist}$ and the server is provided $\genericRand[1]', \dots, \genericRand[\blocklistSize]'$ (\stepsref{authOPRF:auditor:share_random_vals}{authOPRF:auditor:sign_blinding}). The auditors publish $\blocklist'$ and sign (a hash of) $\blocklist'$ under their private keys. The client obtains $\blocklist'$ and the auditors' signatures. After verifying the signatures, the client exponentiates the blinded elements $\{ \left(\oprfRO[1](\genericBlocklistElmt[\setIdx])^{\genericRand[\setIdx]}\right)^{\prfKeyClient}\}_{\genericBlocklistElmt[\setIdx] \in \blocklist}$ and sends them to the server (\stepsref{authOPRF:client:verify_sigs}{authOPRF:client:prf}). Finally, the server removes the blinding and computes $\{ \oprfRO[2](\genericBlocklistElmt[\setIdx], \oprfRO[1](\genericBlocklistElmt[\setIdx])^{\prfKeyClient}\}_{\genericBlocklistElmt[\setIdx] \in \blocklist}$ (\stepref{authOPRF:serverdeblind}).

\subsection{Delegated Fuzzy Private Set Intersection with Associated Data}
%
In this section, we will describe a new primitive called a \dPSIPrimitive (\dPSI). The primitive 
has three parties: a set of auditors \auditor, a server \server and a client \client. Intuitively, 
the auditors publish a set of items $\blocklist$ and want to compute a set intersection between 
$\blocklist$ and a set of items $\cInputSet$ that the client holds. Each item $\genericCInputElmt 
\in \cInputSet$ is a key-value tuple $\tupleDefn{\genericCInputElmt.\keyLabel, 
\genericCInputElmt.\valueLabel}$, where the value is some associated data corresponding to the 
key. The items in set \blocklist do not have any associated data. 
Consider that $\keySetLabel$ is the set of keys and 
$\valSetLabel$ is the set of associated data in $\cInputSet$. 


The intersection is performed on the keys $\keySetLabel \cap \blocklist$ 
and the auditor learns the corresponding associated data from $\valSetLabel$
for each key that is in the intersection. However, the 
auditors are not going to directly run the protocol, and instead delegate the computation to the 
server. The server is provided $\blocklist$ for this purpose. Before describing the requirements, we 
will define the 
threat model. 

\myparagraph{Threat model}

\begin{itemize}
	\item Except for one, all other auditors are untrusted.
	\item The client is honest-but-curious and follows protocols.
	\item The server is untrusted and may collude with the untrusted auditors. 
\end{itemize}


\myparagraph{Functionality requirements}
%
We require the following from the \dPSI functionality. Assume that all
keys in \blocklist and \cInputSet are drawn from a metric space
$(\metricSpace, \distance)$ where $\distance$ is the distance in the

	\item The server can execute the protocol successfully with

	\item Honest auditors are not involved in the computation once

	\item \textbf{Fuzzy matching:} For a public value $\threshold

          server a set $\genericSetVar \subseteq \cInputSet$

	  \item If

            \genericSetVar$ with probability $\ge \tpr$ where $\tpr

	  \item If
            $\distanceNotation[\distance][\genericCInputElmt.\keyLabel][\genericBlocklistElmt]
            > \threshold$, then $\genericCInputElmt \notin
            \genericSetVar$ with probability $\ge \tnr$ where $\tnr
            \in (0,1)$ is the true negative rate.
	  \end{itemize}
\end{itemize}
