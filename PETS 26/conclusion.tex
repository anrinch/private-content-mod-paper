\section{Conclusion}
%
This paper described protocols for detecting harmful content in encrypted files stored on an untrusted server, by comparing against a verified, common blocklist 
published by a distributed set of auditors. When computing exact hash matches against the blocklist, our protocol outperforms the state of the art both in terms of server and client costs. The protocol for fuzzy matching is secure against a malicious server, and features faster compute times on average than the state of the art that operate under a model with a semi-honest server.
